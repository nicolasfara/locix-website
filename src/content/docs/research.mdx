## Research Line

MpaCT is the framework name used on this site for a Scala research line on unifying multiparty distributed programming through capabilities and placement types. The central claim is not only that multitier, choreographic, and aggregate programming can coexist in the same implementation, but that they can be expressed as distinct capability-guarded views over one typed distributed model.

## Terminology

> This site presents the framework as MpaCT; earlier drafts may still use previous names.

This naming update should be stated explicitly whenever older paper text or repository paths still use the previous names. The public site uses MpaCT as the durable identity for the framework.

## What The Work Contributes

1. A unifying design in which multitier, choreographic, and aggregate operations are treated as capabilities rather than unrelated language fragments.
2. Placement types that make location and movement of values explicit across all three paradigms.
3. A typed core formalization with soundness arguments for placed values and inter-paradigm interaction.
4. A Scala reference implementation showing that the design is not only theoretical.
5. An evaluation built around literature-derived use cases plus larger integrated case studies.

## Why Capabilities Matter Here

Capabilities do two jobs at once in this work. First, they delimit where paradigm-specific operations are legal, so a choreographic action is only available inside choreography context, a collective operator inside collective context, and so on. Second, because those capabilities coexist alongside placement types, the language can regulate not only which effects are available but also where values reside and when cross-paradigm access is valid.

That is the positioning difference to emphasize in talks, the homepage, and short project summaries: the contribution is not “a Scala DSL with three feature sets,” but “a typed foundation for combining three distributed paradigms without collapsing them into one vague abstraction.”

## Dissemination Kit

### Tagline

MpaCT is a Scala framework for a capabilities-and-placement-types approach to unifying multitier, choreographic, and aggregate programming.

### Three-Sentence Abstract

MpaCT explores how capabilities and placement types can unify three major multiparty programming styles: multitier programming, choreographic programming, and aggregate computing. The work preserves the distinct abstractions of each paradigm while enabling them to interact safely in one typed distributed model. The result is a Scala reference framework backed by a formal core, compile-time safety arguments, and integrated case studies such as CityEvt and a Nebula port.

### One-Paragraph Summary

Modern distributed applications often need to express architectural placement, protocol coordination, and collective neighborhood behavior in the same system, but current tools usually force those concerns into separate frameworks or ad hoc integration layers. MpaCT explores a different route: use capabilities to delimit paradigm-specific operations and placement types to regulate where values live and how they move, so multitier, choreographic, and aggregate programming can be composed in one typed framework. This site uses MpaCT as the current name for the framework.

### Short Talk Abstract

This talk presents MpaCT, a Scala framework for a broader research line on unifying multitier, choreographic, and aggregate programming through capabilities and placement types. Instead of treating these paradigms as competing languages, the work models them as distinct capability-guarded views over one typed distributed system, making cross-paradigm composition explicit and statically regulated. I will use CityEvt and the Nebula port to show why this matters for real systems that mix placement, protocol structure, and collective behavior.
