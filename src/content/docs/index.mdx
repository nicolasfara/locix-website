## A Shared Foundation

Distributed systems rarely fit a single abstraction.
Real systems mix architectural placement, protocol coordination, and neighborhood-style collective behavior, yet most languages and frameworks force developers to pick one paradigm at a time or stitch several together manually.
{frameworkName} explores a different line: keep those paradigms distinct at the language level, but give them a shared foundation in capabilities and placement types so they can be composed safely.
<Foundations />

## Safety Features

{frameworkName} keeps placement explicit, capabilities scoped, and cross-paradigm handoffs typed, so composition preserves clear boundaries instead of weakening guarantees.
Scala's powerful type system and capture checking make it possible to enforce these properties at compile time, so you can be confident in your code's safety without sacrificing expressiveness.
