## Start Here

If you want to understand the research line quickly, start with the mixed-paradigm examples before the isolated ones.

### Mixed-paradigm examples

- **CityEvt**
  A compact end-to-end example combining multitier registration, choreographic recommendation delivery, and collective crowd monitoring.
  Code: [GitHub](https://github.com/nicolasfara/locix/blob/main/examples/case-studies/city-evt/locix/CityEvt.scala)
- **Nebula port**
  A MpaCT reinterpretation of selected Nebula behavior, combining collective topology awareness, choreographic training coordination, and multitier dashboard reporting.
  Code: [GitHub](https://github.com/nicolasfara/locix/tree/main/nebula/src/io/github/locix/nebula)

### Recommended run commands

```bash
./mill examples.runMain io.github.locix.CityEvt
./mill nebula.runMain io.github.locix.nebula.Nebula
```

## Choreography

Use these when the protocol itself is the main object you want to inspect.

- **PingPong**: minimal communication flow.  
  [GitHub](https://github.com/nicolasfara/locix/blob/main/examples/src/io/github/locix/PingPong.scala)
- **BuyerSellerShipper**: multiparty protocol structure.  
  [GitHub](https://github.com/nicolasfara/locix/blob/main/examples/src/io/github/locix/BuyerSellerShipper.scala)
- **DistributedAuthentication**: security-oriented interaction flow.  
  [GitHub](https://github.com/nicolasfara/locix/tree/main/examples/case-studies/distributed-authentication/locix)
- **MergeSort**: algorithmic decomposition as choreography.  
  [GitHub](https://github.com/nicolasfara/locix/tree/main/examples/case-studies/merge-sort/locix)

## Multitier

Use these when architectural placement and typed cross-tier reads are the main concern.

- **MasterWorker**: explicit task placement across peers.  
  [GitHub](https://github.com/nicolasfara/locix/blob/main/examples/src/io/github/locix/MasterWorker.scala)
- **BasicChat**: simple tiered coordination.  
  [GitHub](https://github.com/nicolasfara/locix/tree/main/examples/case-studies/basic-chat/locix)
- **ChatApplication**: larger multitier communication surface.  
  [GitHub](https://github.com/nicolasfara/locix/tree/main/examples/case-studies/chat-application/locix)
- **TokenRing**: placement-aware distributed control flow.  
  [GitHub](https://github.com/nicolasfara/locix/tree/main/examples/case-studies/token-ring/locix)

## Aggregate

Use these when each node should evolve local state from neighboring context.

- **AggregateCounter**: minimal collective state evolution.  
  [GitHub](https://github.com/nicolasfara/locix/blob/main/examples/src/io/github/locix/AggregateCounter.scala)
- **NeighborDistances**: spatial reasoning over nearby peers.  
  [GitHub](https://github.com/nicolasfara/locix/tree/main/examples/case-studies/neighbor-distances/locix)
- **NeighborDistancesObstacle**: spatial reasoning with obstacles.  
  [GitHub](https://github.com/nicolasfara/locix/tree/main/examples/case-studies/neighbor-distances-obstacle/locix)
- **Broadcasting / Channel / Collection family**: aggregate-style communication and field-oriented patterns.  
  Mapping: [GitHub](https://github.com/nicolasfara/locix/blob/main/examples-mapping.yml)

## Reading Order

1. CityEvt for the overall thesis.
2. One isolated example from each paradigm to learn the local vocabulary.
3. Nebula for the strongest public mixed-paradigm story beyond the introductory case study.
